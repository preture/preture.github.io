# 文件格式概述

在编程与日常工作中，我们会接触到各种各样的文件格式。本文对几种常见的数据与文档格式做一个整体介绍，帮助你在合适的场景选择合适的格式。

## JSON

[JSON](/#/accumulation/programming-foundation/file-formats/json格式说明)（JavaScript Object Notation）是一种轻量级的数据交换格式。

**适用场景：**
- API 请求与响应（RESTful、GraphQL）
- 项目配置文件（`package.json`、`tsconfig.json`）
- Web 数据存储（`localStorage`、`IndexedDB`）
- 序列化与数据传输

**特点：** 语法严格，跨语言支持极好，但缺少注释，不适合手写。

## YAML

[YAML](/#/accumulation/programming-foundation/file-formats/yaml格式说明) 是一种对人类友好的数据序列化格式，常作为配置文件使用。

**适用场景：**
- Docker Compose、Kubernetes 等 DevOps 配置
- GitHub Actions、CI/CD 流水线定义
- 复杂配置（Ansible、Helm）
- 数据定义文件

**特点：** 缩进敏感，可读性极高，支持注释，适合人工编辑的配置文件。

## TOML

[TOML](/#/accumulation/programming-foundation/file-formats/toml格式说明) 是一种旨在成为最小化配置文件格式的语言，强调语义明确。

**适用场景：**
- Python 项目配置（`pyproject.toml`）
- Rust 项目配置（`Cargo.toml`）
- 前端项目配置（`npm` 曾考虑过，`wrangler.toml` 等）
- 需要明确类型和分组的配置文件

**特点：** 语义清晰，表结构明确，歧义最少，适合程序自动读写。

## Markdown

[Markdown](/#/accumulation/programming-foundation/file-formats/markdown格式说明) 是一种轻量级标记语言，专为写作与文档而生。

**适用场景：**
- README 与项目文档
- 博客文章与技术写作
- 笔记与知识管理
- 论坛与即时通讯（Discord、飞书等支持 MD 渲染）

**特点：** 纯文本可读性强，渲染后美观，最适合人类阅读和书写。

## 格式选择指南

| 需求 | 推荐格式 |
|------|---------|
| API 数据交换 | JSON |
| 文档写作 | Markdown |
| 手写配置文件 | YAML / TOML |
| 程序自动读写 | JSON / TOML |
| 复杂嵌套配置 | YAML |
| 最小歧义 | TOML |
